1 To test: * Install tz.cpp by downloading the IANA timezone database
2 at: http://www.iana.org/time-zones You only need the data, not the
3 code.
4 
5 * Change the string `install` in tz.cpp to point to your downloaded
6 IANA database.
7 
8 * Compile validate.cpp along with tz.cpp.
9 
10 * Run the binary and direct the terminal output to a temporary file.
11 
12 * Unzip the tzdata file that has the version corresponding to the IANA
13 database you downloaded (e.g. tzdata2015f.txt.zip).
14 
15 * Compare the unzipped txt file with the output of your validate test
16 program. If they are identical, the test passes, else it fails.
17 
18 Miscellaneous:
19 
20 You can also compare one version of the tzdatabase with another using
21 these uncompressed text files. The text files contain for each
22 timezone its initial state, and each {offset, abbreviation} change
23 contained in the database up through the year 2035. As the database
24 versions change, minor updates to the set of these transitions are
25 typically made, typically due to changes in government policies.
