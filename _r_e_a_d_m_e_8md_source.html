15 [Causal Dynamical Triangulations][CDT] in [C++] use the
16 [Computational Geometry Algorithms Library][CGAL], [Boost], [TBB], and [Eigen].
17 Arbitrary-precision numbers and functions via [MPFR] and [GMP].
18 Uses [Docopt] to provide a beautiful command-line interface, and [Howard Hinnant's date and timezone][date] library for accurate time.
19 Uses [Catch] for [BDD]/[TDD].
20 Uses [Conan]/[CMake]/[Ninja] for dependency management and building.
21 Uses [Doxygen] for automated document generation.

76 ~~~

80 This finds your compiler settings, and allows the [build.sh] script to run correctly.

84 ~~~

86 ~~~

88 Using [CMake] and [Ninja], Conan handles most of the dependencies, except [CGAL] and [TBB]. The easiest way to
89 install these so that Conan can find them is with [Homebrew] or [Linuxbrew]. Once these are installed:

91

105 ### Installing
106 
107 This project uses a separate `build/` directory, which allows you to rebuild the
108 project without cluttering the source code. Thus, download this source code
109 (clone this repo from GitHub or grab a [release]) and run the [build.sh] script.
110 
111 This should result in the main program executable, `cdt` in the `build/bin`
112 directory, along with several others.
113 
114 * `CDT_test` is the [Catch] executable which runs the unit tests
115 * `cdt-gv` converts output files to [Geomview] format for visualization
116 * `cdt-opt` is a simplified version with hard-coded inputs, mainly useful for
117 debugging and scripting
118 * `initialize` is used by [CometML] to run [parameter optimization](#parameter-optimization)
119 
120 The install script will also run unit and integration tests.
121 See [Testing](#testing) for details.

123 ### Usage
124 
125 CDT-plusplus uses [Docopt] to parse options from the help message, and so
126 understands long or short argument formats, provided the short argument given
127 is an unambiguous match to a longer one. The help message should be instructive:
128 
129 ~~~
130 ./build/cdt --help
131 Causal Dynamical Triangulations in C++ using CGAL.
132 
133 Copyright (c) 2014-2018 Adam Getchell
134 
135 A program that generates d-dimensional triangulated spacetimes
136 with a defined causal structure and evolves them according
137 to the Metropolis algorithm. Specify the number of passes to control
138 how much evolution is desired. Each pass attempts a number of ergodic
139 moves equal to the number of simplices in the simulation.
140 
141 Usage:./cdt (--spherical | --toroidal) -n SIMPLICES -t TIMESLICES [-d DIM] -k K --alpha ALPHA --lambda LAMBDA [-p PASSES] [-c CHECKPOINT]
142 
143 Examples:
144 ./cdt --spherical -n 32000 -t 11 --alpha 0.6 -k 1.1 --lambda 0.1 --passes 1000
145 ./cdt --s -n32000 -t11 -a.6 -k1.1 -l.1 -p1000
146 
147 Options:
148  -h --help Show this message
149  --version Show program version
150  -n SIMPLICES Approximate number of simplices
151  -t TIMESLICES Number of timeslices
152  -d DIM Dimensionality [default: 3]
153  -a --alpha ALPHA Negative squared geodesic length of 1-d
154  timelike edges
155  -k K K = 1/(8*pi*G_newton)
156  -l --lambda LAMBDA K * Cosmological constant
157  -p --passes PASSES Number of passes [default: 100]
158  -c --checkpoint CHECKPOINT Checkpoint every n passes [default: 10]
159 ~~~
160 
161 The dimensionality of the spacetime is such that each slice of spacetime is
162 `d-1`-dimensional, so setting `d=3` generates 2 spacelike dimensions and one
163 timelike dimension, with a defined global time foliation. Thus a
164 `d`-dimensional simplex will have some `d-1` sub-simplices that are purely
165 spacelike (all on the same timeslice) as well as some that are timelike
166 (span two timeslices). In [CDT] we actually care more about the timelike
167 links (in 2+1 spacetime) and the timelike faces (in 3+1 spacetime).

169 ## Documentation
170 
171 Online documentation may be found at http://www.adamgetchell.org/CDT-plusplus/
172 
173 If you have [Doxygen] installed you can generate the same information
174 locally by simply typing at the top level directory
175 ([Doxygen] will recursively search):
176 
177 ~~~
178 doxygen
179 ~~~
180 
181 This will generate the `html/` directory which will contain
182 documentation generated from CDT++ source files. `USE_MATHJAX` has been enabled
183 in [Doxyfile] so that the LaTeX formulae can be rendered in the html
184 documentation using [MathJax]. `HAVE_DOT` is set to **YES** which allows
185 various graphs to be autogenerated by [Doxygen] using [GraphViz].
186 If you do not have GraphViz installed, set this option to **NO**
187 (along with `UML_LOOK`).

247 ## Parameter Optimization
248 
249 [CometML] is used to record [Experiments] which conduct [Model Optimization]. The script to do
250 this is `optimize-initialize.py`. In order for this to work, you must install the following
251 into your Python [virtual environment].
252 
253 ~~~
254 pip install tensorflow
255 pip install comet-ml
256 ~~~
